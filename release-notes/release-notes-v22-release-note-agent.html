Implementation Details

The new system follows these steps:

  1. Reads the existing index.html from the public_repo directory
  2. Parses the HTML structure to understand the current layout
  3. Extracts existing release cards from the releases-grid div
  4. Creates a new release card using the same format as existing cards
  5. Updates the "Latest" badge appropriately
  6. Sorts releases by version (newest first)
  7. Updates the footer timestamp
  8. Saves the updated index.html to the release-notes directory
